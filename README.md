# ShellQuicker

> 一个用 Godot 4 做的 GUI 工具，帮你把复杂的命令行指令变成可维护的「参数模板」

---

## 解决的问题

当你需要频繁使用 FFmpeg、yt-dlp、curl 等命令行工具时，经常会面临：
- 参数太长太乱，难以复用
- 记不清某个参数的含义
- 每次都要从笔记里复制粘贴

ShellQuicker 把这些「烂命令」变成可视化的「参数卡片」，每个参数独立一行，可加中文备注，还能一键执行。

---

## 快速上手

### 基本操作流
1. **新建模板** → 输入核心命令（如 `ffmpeg -i`）
2. **智能解析** → 粘贴完整命令，一键拆分成参数行
3. **添加备注** → 为每个参数写上中文说明
4. **保存执行** → 直接运行或复制完整指令

### 快捷键

| 按键 | 作用 |
|------|------|
| `Ctrl+N` | 新建文件夹 |
| `Ctrl+A` | 新建模板 |
| `Ctrl+D` | 克隆当前模板 |
| `Ctrl+B` | 添加新参数行 |
| `Ctrl+G` | 合并选中的参数行 |
| `Ctrl+Enter` | 强制执行当前命令 |
| `Shift+Enter` | 智能解析命令 |
| `F2` / 双击 | 重命名 |
| `Enter` | 执行选中项 |
| `Delete` | 删除 |

## 功能特性

### 命令解析
- **智能解析**：粘贴任意长命令，自动拆分 `-flag value` 结构
- **手动合并**：解析不理想？用 `Ctrl+G` 手动合并参数
- **参数重排**：拖拽调整组装顺序

### 多配置文件
- 支持多个 `.json` 文件切换工作区
- 适合按项目或用途分类管理

### 执行终端
- 内置 4 种执行方式：CMD、PowerShell、Windows Terminal (CMD/PS)
- UTF-8 模式 (`chcp 65001`) 解决中文路径乱码

### 单实例
- 默认只允许运行一个实例（端口 58123 互斥）
- 底部可勾选「允许多开」解除限制

## 项目结构

```
ShellQuicker/
├── 入口.gd           # 主 UI 逻辑 (~1000行，纯代码生成控件)
├── 配置管理器.gd    # Autoload 单例，管理配置持久化
├── 树辅助.gd        # 树形控件拖拽逻辑
├── 主界面.tscn      # 场景入口
├── presets.json     # 默认数据文件
├── settings.cfg    # 全局设置 (字体大小、分隔偏移等)
└── project.godot   # Godot 项目配置
```

### 数据存储
- **配置路径**: exe 所在目录
- **全局设置**: `settings.cfg`
- **模板数据**: `*.json`（默认 `presets.json`）

---

## 开发相关

- **引擎**: Godot 4.6 (Forward Plus)
- **运行**: 直接用 Godot 打开目录，或运行导出后的 exe
- **UI 生成**: 纯代码创建，无 `.tscn` 控件定义
- **调试**: 项目使用 `print()` 输出，可保留
- **导出**: `export_presets.cfg` 已配置 Windows 导出
