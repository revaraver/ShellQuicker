# ShellQuicker

## 1. 项目简介 (Introduction)
ShellQuicker 是一款基于 Godot 4 引擎开发的轻量级带图形界面的命令行参数管理工具。
主要设计目标是解决复杂 CLI（命令行接口）工具（如 FFmpeg、ImageMagick、yt-dlp 等）长参数难以记忆、难以阅读和难以复用的问题。通过将传统的单行长命令碎片化为可视化的“键值对”结构，并为每个参数独立提供中文字段注释功能，从而降低开发与使用过程中的认知负担。

## 2. 核心功能与特性 (Core Features)
- **参数原子化与可视化管理**：支持将单行命令拆分为紧凑垂直排列的参数行。开发者可为每一个参数（或标志位）添加独立的说明，不再依赖外部文档或文本文件中的混乱注释。
- **动态预览与高亮映射**：对参数或核心指令进行编辑时，底部预览区会实时生成最终组装出的完整命令行。系统会以纯绿色高亮当前聚焦的参数部分，保持视觉同步。
- **基于 TCP 端口的单实例控制机制**：默认通过监听本地 58123 端口实现硬防多开，以此取代不可靠的引擎原生方法。同时支持通过 UI 设置项即时切换，开放多实例并行运行。
- **UTF-8 环境适配**：内置 `chcp 65001` 指令注入选项。一键解决 Windows 默认 CMD 或 PowerShell 在执行带有中文字符路径的复杂命令时频发的输出乱码问题。
- **本地化与低占用率**：相比基于 Electron 框架或需要后台常驻 Python Kernel 的 Web 式交互环境（如 Jupyter），本工具依托 C++ 底层渲染框架作为界面载体，具有零依赖和瞬时启动的特点。在复杂渲染下的内存占用量稳定在 200MB 左右。

## 3. 快捷键与操作指南 (Shortcuts & Usage)

### 目录区操作（左侧面板）
不论光标目前处于何处，以下全局快捷键对操作树文件稳定生效：
- **`Ctrl + N`**：在当前层级新建文件夹。
- **`Ctrl + A`**：在当前层级新建一条空白命令模板。
- **`Ctrl + D`**：克隆当前选中的命令模板（连同其下属的全部配置项和参数列表一并生成副本）。
- **`F2`** 或 **双击节点**：重命名选中项。
- **`Delete`**：删除所选中的模板或文件夹。
- **`Enter`** / **`Ctrl + Enter`**：直接执行当前选中的命令配置（仅在焦点不处于输入框时触发有效）。
- **鼠标拖拽**：支持单选/多选拖拽排布顺序，并允许拖入文件夹内以构建多层树状嵌套逻辑。

### 参数区交互（右侧上方）
- **长命令快速拆解**：在“核心命令”栏粘贴一长段原始 CLI 指令后，点击右侧【智能解析】按钮，系统将按照语法自动抽离并向下排布子级参数序列。
- **`Ctrl + B`**：在当前选中的参数行正下方，立即插入一条全新的参数空位。
- **拖拽重排**：按住单行参数最左侧的 `☰` 图标，可以将其上下拖动。这会同步改变该参数拼接进最终命令文本中的实际语法顺序。

### 预览组装区（右侧下方）
- **合并选区 (`Ctrl + G`)**：使用鼠标在底部的原生命令行预览面板中，框选连续的某一段参数文本（例如 `-resize 256x256`）。随后按下 `Ctrl + G`。程序会自动寻找上方列表中与之对应的部分，并将它们物理合并为单独的一行参数，方便整体搬移。
- **反向悬停定位 (`G` 键)**：在未框选任何内容时，将鼠标指针悬停在底部命令行预览面板的任一段落上，按下 `G` 键。系统将通过鼠标坐标推断其所属指令的本体，并在上方参数编辑列中自动将其聚焦高亮，实现参数的快速反查。
- **纯净剪贴板**：预览区域中无论是否带有颜色高亮，直接拖动选定的文本并使用 `Ctrl + C`，其复制出的内容均不附带任何内部转义符污染，粘贴至任何终端皆可直接运行。

## 4. 技术实现说明 (Tech Notes)
**关于 TCPServer 多实例判定**：
Godot 引擎在 Windows 环境下的多开阻止和外部参数传递，通常面临无法跨系统边界稳定运行的缺陷。因此我们在 `配置管理器.gd` 中手动引入了更稳健的服务器占用策略：在启动周期更早期通过抢占式监听特定端口（58123）来实现本地进程锁。
如用户需要同时拉取多个独立进程（例如比对多组复杂参数的渲染差异），可直接在软件底部的 UI 中勾选【允许多开】。该状态变更将直接写入 `settings.cfg` 持久化配置文件，并在下次克隆运行进程时自动执行旁路跳过。
